## О проекте

Данный репозиторий содержит решение тестового задания для позиции **junior-системный аналитик** в интернет-магазин «Петрушка Зеленая».

В рамках задания выполнены:
- Анализ требований к функционалу «Корзина»
- Проектирование REST API для экрана магазинов-партнеров
- Разработка верхнеуровневой архитектуры push-уведомлений

## Структура репозитория

|------|----------|

## Содержание заданий

### Задание 1. Анализ требований (Корзина)
- Выявлены логические противоречия и недочеты исходного ТЗ
- Предложена исправленная версия ТЗ с 13 пунктами
- Сформулированы уточняющие вопросы продукт-менеджеру (10 вопросов)

### Задание 2. Проектирование REST API
- Разработан REST API для экрана выбора магазинов-партнеров
- Метод: `GET /api/v1/partner-shops`
- API протестирован в Postman с использованием Mock Server

### Задание 3. Архитектура push-уведомлений
- Разработана верхнеуровневая архитектурная схема
- Асинхронная микросервисная архитектура с использованием Kafka/RMQ
- Отдельный сервис уведомлений с отправкой через FCM (Android) и APNs (iOS)
